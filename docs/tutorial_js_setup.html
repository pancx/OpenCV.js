Build OpenCV.js

Installing Emscripten

Emscripten is an LLVM-to-JavaScript compiler. We will use Emscripten to build OpenCV.js.

To Install Emscripten, follow instructions on Emscripten SDK.

For example

1 ./emsdk update
2 ./emsdk install latest
3 ./emsdk activate latest
Note
To compile to WebAssembly, you also need to install and activate Binaryen with the emsdk command. Please refer to Developer's Guide for more details.

After install, ensure the EMSCRIPTEN environment is setup correctly.

For example:

1 source ./emsdk_env.sh
2 echo ${EMSCRIPTEN}

Obtaining OpenCV Source Code

You can use the latest stable OpenCV version or you can grab the latest snapshot from our Git repository.

Obtaining the Latest Stable OpenCV Version

  • Go to our downloads page.
  • Download the source archive and unpack it.

Obtaining the Cutting-edge OpenCV from the Git Repository

Launch Git client and clone OpenCV repository.

For example:

1 cd ~/<my_working_directory>
2 git clone https://github.com/opencv/opencv.git
Note
You may need to install git for your development environment.

Building OpenCV.js from Source Using CMake

  1. Create and open a temporary directory <cmake_build_dir> (build_js in this example), put the generated Makefiles, project files, and output files.

    For example:

    1 cd ~/opencv
    2 mkdir build_js
    3 cd build_js

  2. To configure, run cmake [<some optional parameters>] <path to the OpenCV source directory> To build OpenCV.js, you need to append -D CMAKE_TOOLCHAIN_FILE=${EMSCRIPTEN}/cmake/Modules/Platform/Emscripten.cmake.

    For example:

    1 cmake -D CMAKE_TOOLCHAIN_FILE=${EMSCRIPTEN}/cmake/Modules/Platform/Emscripten.cmake -D CMAKE_BUILD_TYPE=Release -D CMAKE_INSTALL_PREFIX=/usr/local ..
    Note
    You may need to install cmake for your development environment.
    Use cmake -DCMAKE_TOOLCHAIN_FILE=${EMSCRIPTEN}/cmake/Modules/Platform/Emscripten.cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr/local .. , without spaces after -D if the above example doesn't work.
    Pass -DCMAKE_C_FLAGS="-s WASM=1" and -DCMAKE_CXX_FLAGS="-s WASM=1" to cmake if you are targeting WebAssembly.

  3. To build, run make with target "opencv.js" from the build directory. It is recommended to do this in several threads.

    For example:

    1 make -j7 opencv.js # runs 7 jobs in parallel

    The opencv.js found <cmake_build_dir>/bin folder is the final product to include into your web pages.

  4. [optional] To build documents, run make with target "doxygen"

    For example:

    1 make doxygen

    The built documents are located in the <cmake_build_dir>/doc/doxygen/html folder.

    Note
    You may need to install doxygen tool for your development environment.

  5. [optional] To run tests, first run make with target "opencv_js_test"

    For example:

    1 make opencv_js_test

    Then run a local web server in <cmake_build_dir>/bin folder. For example, node http-server which serves on localhost:8080.

    Navigate the web browser to http://localhost:8000/tests.html, which runs the unit tests automatically.

    You can also run tests using Node.js.

    For example:

    1 cd bin
    2 npm install
    3 node tests.js
    Note
    You may need to install node for your development environment.
